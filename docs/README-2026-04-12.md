# Registro de tareas — 2026-04-12

## Objetivo del día
Sincronizar ramas locales con GitHub y montar el pipeline de auto-deploy en VPS con Watchtower + GHCR.

## Tareas realizadas

1. Sincronización de ramas locales
   - `main` local estaba 52 commits por detrás de `origin/main` (incluía el PR #18 de leandroPII → main).
   - `leandroPII` local estaba 9 commits por detrás de `origin/leandroPII`.
   - Se ejecutó `git pull` en ambas ramas. Fast-forward limpio en los dos casos.

2. Diagnóstico del pipeline roto
   - La VPS usaba imagen local `apalabrazos-leandropii:latest` (cargada manualmente el 2026-04-02).
   - GitHub Actions publica en `ghcr.io/trutoman/apalabrazos:latest`.
   - No había login a GHCR en la VPS ni Watchtower corriendo — el auto-deploy nunca funcionó.

3. Login a GHCR en la VPS
   - Se generó un PAT con scope `read:packages`.
   - Se ejecutó `docker login ghcr.io -u trutoman` en la VPS con el token.
   - Credenciales almacenadas en `/root/.docker/config.json`.

4. Actualización de `docker-compose.yml` en VPS
   - Se cambió la imagen del backend de `apalabrazos-leandropii:latest` a `ghcr.io/trutoman/apalabrazos:latest`.
   - Se añadió el servicio `watchtower` al compose.

5. Watchtower configurado
   - Imagen: `containrrr/watchtower`.
   - Monitorea únicamente el contenedor `apalabrazos-backend`.
   - Poll cada 300 segundos (5 minutos).
   - Limpieza automática de imágenes viejas (`WATCHTOWER_CLEANUP=true`).
   - Usa las credenciales de GHCR vía volumen `/root/.docker/config.json`.

6. Despliegue validado
   - `docker compose up -d` aplicado correctamente.
   - Backend respondiendo HTTP 200 en `http://localhost:8080/`.
   - `https://apalabrazos.app` operativo.

## Resultado
- Pipeline de auto-deploy completamente funcional.
- Push a `main` que toque `src/**`, `Dockerfile` o `pom.xml` dispara GitHub Actions → publica imagen en GHCR → Watchtower la detecta en ≤5 min → reinicia el backend automáticamente.
- `leandroPII` es rama de trabajo — no dispara el pipeline de producción.

## Corrección de workflow

- El workflow de GitHub Actions estaba configurado para dispararse desde `main` **y** `leandroPII`.
- Ambas ramas publicaban `:latest`, lo que significaba que cualquier push de trabajo a `leandroPII` pisaba la imagen de producción.
- Se corrigió `.github/workflows/deploy-backend.yml` para que solo se dispare desde `main`.
- Flujo correcto: `leandroPII → PR → main → Actions → GHCR :latest → Watchtower → VPS`.
